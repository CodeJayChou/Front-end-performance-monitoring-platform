PHASE 2 — Client 成为唯一真入口
2.1 事件归属统一
行动目标

确保只有 Client.capture() 能触发事件流

必须迁移逻辑

全部收敛到 Client：

integration 执行
Hub/Scope 注入
middleware 执行
beforeSend
transport 发送
代码目标结构
capture(event) {
  event = this.runIntegrations(event)
  event = this.applyHubScope(event)
  event = this.middleware.execute(event)
  event = this.beforeSend(event)

  if (!event) return

  return this.transport.send(event)
}

PHASE 3 — Middleware 系统重构（关键架构整理）
3.1 middleware 分类

新增明确分类：

enum MiddlewareType {
  STRUCTURAL = "structural",   // 结构化：normalize
  CONTEXTUAL = "contextual",   // 上下文：enrich
  POLICY = "policy"            // 策略：filter/sample
}
3.2 middleware 标记
对应关系：
normalize → STRUCTURAL
enrich → CONTEXTUAL
filter / sample → POLICY
3.3 执行顺序标准化

替代“纯 priority 模型”：

STRUCTURAL → CONTEXTUAL → POLICY

同组内：

priority 从高到低
3.4 调度实现调整
sort(mw) {
  return mw.sort((a, b) =>
    a.type !== b.type
      ? typeOrder[a.type] - typeOrder[b.type]
      : b.priority - a.priority
  )
}

PHASE 5 — Event Contract 收敛（类型稳定化）

5.2 BaseEvent 调整

interface BaseEvent<T = unknown> {

  type: EventType
  timestamp: number
  platform: string

}
5.3 增加验证层
